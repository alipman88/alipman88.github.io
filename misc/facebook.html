Processing externally-stored images with HTML5 & uploading base64 URI's to Facebook
Aaron Lipman, June 2013

I recently wrote code for a really interesting project that involved using Facebook's API to grab profile pics, apply custom image filtering and text, and post the pictures on users' timelines.

See it in action | View the source

While Rails (my framework of choice) offers tons of convenient options via gems like fog and CarrierWave, and command-line utilities like ImageMagick, I chose to write as much of the code on the client-side as I could, using AJAX to handle API integration, and an HTML5 canvas to apply image processing.

I made this decision reasoning that this was a perfect opportunity to familiarize myself with HTML5's canvas element, and also because using a light-weight browser-run solution to handle API integration feels like the way the web is *supposed* to work. (Also, in my freelance work, I handle a lot of requests for social media integration -- writing code in JavaScript allows me to efficiently repurpose my code, regardless of the native programming language/framework used by the organization I'm contracting for.)

However, this presented a couple challenges, some of which weren't immediately apparent. First-off was posting images to Facebook from HTML5 canvasses; Facebook doesn't provide a means of uploading images from base64 data URI's (the storage method used by canvas elements).

The solution? Use imgur's API as a middle-layer; send the base64 data to imgur as an upload, and pass the URL created by imgur to Facebook's API:

This worked for about 48 hours. Confused, I pulled up Chrome's JavaScript console, and found that my image processing function, specifically the toDataURL() line was throwing a security error. Apparently, Facebook either changed their CORS policy overnight, or flagged my app/website for sending to many image requests with crossOrigin = "Anonymous";, and stopped providing me with CORS-enabled images.

(In non-technical terms, this meant Facebook would let my script load images, but not allow the image data to be read via an HTML5 canvas -- I could *see* the images, but I couldn't *do* anything with them.)

Easy fix? Use a proxy that allows access to external resources that would otherwise be blocked due to a same-origin policy. The most convenient I know is corsproxy.com:

This worked great! However there were still a couple small issues. Imgur's API provides a max of only 1250 free uploads a day, per app. Also, why depend on a third-party service for handling CORS requests? In the end, since the project was going to be a Rails app hosted on Heroku, I built my own lightweight image-proxy, and used Amazon Web Service's S3 hosting instead of imgur as my middle layer, sending my images as BLOB's. (See aws.amazon.com/articles/1434 for more on this. You'll need a server-side method to encrypt your AWS secret key.)
